## 🤖 AI-Assisted Workflow (Agentic Dev Stack)

Following the **"No Manual Coding"** rule, I implemented the deployment using this flow:

```
Write Spec → Generate Plan → Task Breakdown → Implementation via AI Agents
```

### 1. Containerization with Gordon

I used **Docker AI (Gordon)** to generate optimized Dockerfiles for the frontend and backend.

```bash
docker ai "Create a multi-stage Dockerfile for my React frontend and FastAPI backend"
```

- Backend: `python:3.11-slim` — 2-stage build, non-root user, port 8000
- Frontend: `node:18-alpine` — 3-stage build with Next.js standalone output, port 3000

### 2. Kubernetes Manifests with kubectl-ai

Instead of writing YAML manually, I used **kubectl-ai** to generate all Kubernetes resources.

```bash
kubectl-ai "deploy the todo frontend with 2 replicas and a LoadBalancer service"
```

Generated manifests:
- `backend-deployment.yaml` — 2 replicas, liveness/readiness probes, resource limits
- `frontend-deployment.yaml` — 2 replicas, env from ConfigMap
- `backend-service.yaml` — ClusterIP (internal DNS)
- `frontend-service.yaml` — NodePort (external browser access on port 30080)
- `configmap.yaml` — `NEXT_PUBLIC_API_URL=http://todo-backend:8000`
- `pvc.yaml` — 1Gi PersistentVolumeClaim for data durability

### 3. Cluster Optimization with Kagent

I utilized **Kagent** to ensure cluster health and optimal resource allocation.

```bash
# Pass 1 — stability check
kagent "analyze the cluster health and suggest resource limits"

# Pass 2 — resource right-sizing
kagent "Analyze CPU and memory utilization for todo-frontend and todo-backend pods"
```

## 📦 Deployment Instructions

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (4.53+ with Gordon enabled)
- [Minikube](https://minikube.sigs.k8s.io/docs/start/) installed and in PATH
- [Helm 3](https://helm.sh/docs/intro/install/) installed
- [kubectl-ai](https://github.com/GoogleCloudPlatform/kubectl-ai) plugin configured
- [Kagent](https://kagent.dev/) installed

### Environment Variables

Create a `.env` file at the repo root (never commit this):

```env
DATABASE_URL=postgresql+asyncpg://<user>:<password>@<host>/<db>?ssl=require
OPENAI_API_KEY=sk-...
BETTER_AUTH_SECRET=<your-secret>
```

### Steps to Run Locally

**1. Start Minikube**

```bash
minikube start
minikube addons enable dashboard
minikube addons enable metrics-server
```

**2. Switch Docker context to Minikube daemon**

```bash
eval $(minikube docker-env)
```

> ⚠️ Run this in every new terminal session before building images.

**3. Create Kubernetes Secret & ConfigMap**

```bash
kubectl create secret generic todo-backend-secret \
  --from-literal=DATABASE_URL="$DATABASE_URL" \
  --from-literal=OPENAI_API_KEY="$OPENAI_API_KEY" \
  --from-literal=BETTER_AUTH_SECRET="$BETTER_AUTH_SECRET"

kubectl create configmap todo-frontend-config \
  --from-literal=NEXT_PUBLIC_API_URL="http://todo-backend:8000"
```

**4. Build & Load Images**

```bash
docker build -t todo-backend:latest -f Dockerfile.backend .
docker build -t todo-frontend:latest -f Dockerfile.frontend ./frontend
```

**5. Deploy via Helm**

```bash
helm install todo-chatbot ./charts/todo-app
```

**6. Verify Status**

```bash
kubectl get pods        # All 4 pods should be Running/Ready (1/1)
kubectl get services    # frontend NodePort 30080, backend ClusterIP
kubectl get pvc         # todo-data-pvc should be Bound
```

**7. Access the App**

```bash
minikube service todo-frontend
```

> Opens the Todo Chatbot UI in your default browser automatically.

### Teardown

```bash
helm uninstall todo-chatbot
minikube stop
```
